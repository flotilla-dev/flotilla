# Package Compatibility Testing Guide

This guide shows you how to verify package compatibility before and after installation.

## Method 1: pip-tools (Recommended)

### Install pip-tools
```bash
pip install pip-tools
```

### Check Dependencies
```bash
# Create requirements.in with your main packages
cat > requirements.in << EOF
langchain==1.0.1
langchain-openai==1.0.1
langchain-community==0.4.0
pydantic==2.12.2
pydantic-settings==2.11.0
EOF

# Compile with all sub-dependencies
pip-compile requirements.in

# This will:
# 1. Resolve all dependencies
# 2. Show version conflicts if any
# 3. Generate requirements.txt with pinned versions
```

**Output Example:**
```
# This file is autogenerated by pip-compile
langchain==1.0.1
  via -r requirements.in
langchain-community==0.4.0
  via -r requirements.in
langchain-core==0.4.0
  via langchain, langchain-community, langchain-openai
langchain-openai==1.0.1
  via -r requirements.in
pydantic==2.12.2
  via -r requirements.in
```

If there are **conflicts**, you'll see errors like:
```
ERROR: Cannot install langchain==1.0.1 and langchain-core==0.3.0 because these package versions have conflicting dependencies.
```

## Method 2: pipdeptree

### Install pipdeptree
```bash
pip install pipdeptree
```

### View Dependency Tree
```bash
# After installing your packages
pipdeptree

# Example output:
langchain==1.0.1
  - langchain-core [required: ^0.4.0, installed: 0.4.0]
  - langchain-text-splitters [required: ^0.4.0, installed: 0.4.0]
langchain-openai==1.0.1
  - langchain-core [required: ^0.4.0, installed: 0.4.0]
  - openai [required: ^1.54.0, installed: 1.54.3]
pydantic==2.12.2
  - pydantic-core [required: ==2.27.2, installed: 2.27.2]
```

### Find Conflicts
```bash
# Check for version conflicts
pipdeptree --warn warn

# Reverse dependency tree (who depends on X?)
pipdeptree -r -p langchain-core

# Output shows which packages need langchain-core:
langchain-core==0.4.0
  - langchain==1.0.1 [requires: langchain-core^0.4.0]
  - langchain-openai==1.0.1 [requires: langchain-core^0.4.0]
  - langchain-community==0.4.0 [requires: langchain-core^0.4.0]
```

## Method 3: pip check

### Quick Conflict Check
```bash
# Install packages first
pip install -r requirements.txt

# Check for conflicts
pip check
```

**Success Output:**
```

```

**Conflict Output:**


## Method 4: Poetry (Modern Alternative)

### Setup Poetry
```bash
# Install Poetry
curl -sSL https://install.python-poetry.org | python3 -

```

### Add Dependencies
```bash

poetry add langchain-openai@1.0.1

poetry add pydantic@2.12.2
```

- Resolves dependencies
- Detects conflicts
- Creates lock file with exact versions
- Shows clear error messages for incompatibilities

## Pre-Installation Compatibility Check Script

Save this as `check_compatibility.py`:

```python
#!/usr/bin/env python3
"""
Check package compatibility before installation
"""
import subprocess
import sys
import json

def check_compatibility():
    packages = [
        "langchain==1.0.1",
        "langchain-openai==1.0.1",
        "langchain-community==0.4.0",
        "pydantic==2.12.2",
        "pydantic-settings==2.11.0"
    ]
    
    print("🔍 Checking package compatibility...\n")
    
    # Use pip's dependency resolver in dry-run mode
    cmd = ["pip", "install", "--dry-run", "--report", "-"] + packages
    
    try:
        result = subprocess.run(
            cmd,
            capture_output=True,
            text=True,
            check=True
        )
        
        # Parse the report
        report = json.loads(result.stdout)
        
        print("✅ All packages are compatible!\n")
        print("Would install:")
        for item in report.get("install", []):
            metadata = item.get("metadata", {})
            name = metadata.get("name", "unknown")
            version = metadata.get("version", "unknown")
            print(f"  - {name}=={version}")
            
        return True
        
    except subprocess.CalledProcessError as e:
        print("❌ Compatibility issues found:\n")
        print(e.stderr)
        return False
    except json.JSONDecodeError:
        print("⚠️  Could not parse pip output")
        print("Try manual installation to see detailed errors")
        return False

if __name__ == "__main__":
    success = check_compatibility()
    sys.exit(0 if success else 1)
```

Run it:
```bash
python check_compatibility.py
```

## Quick Compatibility Test After Installation

Create `test_compatibility.py`:

```python
#!/usr/bin/env python3
"""Test that all packages work together"""

def test_imports():
    """Test all critical imports"""
    print("Testing imports...")
    
    try:
        # Core LangChain
        import langchain
        print(f"✅ langchain {langchain.__version__}")
        
        # OpenAI integration
        from langchain_openai import AzureChatOpenAI
        print(f"✅ langchain-openai imported")
        
        # Community features
        import langchain_community
        print(f"✅ langchain-community imported")
        
        # Core modules
        from langchain_core.messages import HumanMessage, SystemMessage
        from langchain_core.tools import StructuredTool
        from langchain_core.prompts import ChatPromptTemplate
        print(f"✅ langchain-core modules imported")
        
        # Pydantic
        from pydantic import BaseModel, Field
        from pydantic_settings import BaseSettings
        print(f"✅ pydantic and pydantic-settings imported")
        
        return True
        
    except ImportError as e:
        print(f"❌ Import failed: {e}")
        return False

def test_basic_functionality():
    """Test basic functionality"""
    print("\nTesting basic functionality...")
    
    try:
        from pydantic import BaseModel, Field
        
        class TestModel(BaseModel):
            name: str = Field(..., description="Test field")
            value: int = 42
        
        model = TestModel(name="test")
        assert model.value == 42
        print("✅ Pydantic models work")
        
        from langchain_core.messages import HumanMessage
        msg = HumanMessage(content="test")
        assert msg.content == "test"
        print("✅ LangChain messages work")
        
        return True
        
    except Exception as e:
        print(f"❌ Functionality test failed: {e}")
        return False

if __name__ == "__main__":
    import sys
    
    imports_ok = test_imports()
    functionality_ok = test_basic_functionality()
    
    if imports_ok and functionality_ok:
        print("\n🎉 All compatibility tests passed!")
        sys.exit(0)
    else:
        print("\n❌ Some tests failed")
        sys.exit(1)
```

Run it:
```bash
python test_compatibility.py
```

## Recommended Workflow

1. **Before changing versions:**
   ```bash
   # Check current state
   pip freeze > before_requirements.txt
   pipdeptree > before_tree.txt
   ```

2. **Test new versions in isolation:**
   ```bash
   python -m venv test_env
   source test_env/bin/activate
   pip install -r requirements.txt
   python test_compatibility.py
   ```

3. **If successful, update main environment:**
   ```bash
   deactivate
   pip install -r requirements.txt --upgrade
   python test_compatibility.py
   ```

4. **Document final versions:**
   ```bash
   pip freeze > final_requirements.txt
   pipdeptree > final_tree.txt
   ```

## Common Issues and Solutions

### Issue: "ERROR: pip's dependency resolver does not currently take into account..."

**Solution:** Use `pip-compile` or Poetry which have better resolvers.

### Issue: Multiple versions of same package

**Solution:**
```bash
pip uninstall <package> -y
pip install <package>==<version>
```

### Issue: Cached packages causing issues

**Solution:**
```bash
pip cache purge
pip install -r requirements.txt --no-cache-dir
```
